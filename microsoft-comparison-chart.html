Detailed Technology Information

M365 Copilot API

When to Use ‚úì

  • Need semantic search over SharePoint/OneDrive content
  • Want conversational AI with M365 data grounding
  • Require meeting insights and AI summaries
  • Building apps that leverage M365 Copilot intelligence
  • Need permission-aware search results

When NOT to Use ‚úó

  • Building a custom agent (use M365 Agents SDK)
  • Need multi-agent orchestration (use Agent Framework)
  • Only need bot messaging (use DirectLine API)
  • Require on-premises deployment

Key APIs

  • Retrieval API: Semantic search with permission trimming
  • Chat API: Multi-turn conversations with M365 grounding
  • Meeting Insights API: AI-generated meeting summaries
  • Search API: Semantic search across OneDrive

DirectLine API

When to Use ‚úì

  • Have an existing Bot Framework bot
  • Need a custom chat UI (web, mobile, desktop)
  • Want full control over client experience
  • Require cross-platform bot integration

When NOT to Use ‚úó

  • Building new agent for M365/Teams (use M365 Agents SDK)
  • Need M365-specific features (use Agent 365 SDK)
  • Need multi-agent orchestration (use Agent Framework)
  • Starting from scratch without Bot Framework

Key Features

  • Standard HTTPS protocol with WebSocket support
  • Token-based authentication
  • Client libraries for C# and Node.js
  • Compatible with Bot Framework SDK 4.0

M365 Agents SDK

When to Use ‚úì

  • Building a new conversational agent
  • Want multi-channel deployment (Teams, M365, Web)
  • Need unified scaffolding for agent communication
  • Want AI-service flexibility (any LLM provider)
  • Prefer unified codebase across platforms

When NOT to Use ‚úó

  • Need advanced M365 integration (use Agent 365 SDK)
  • Only accessing M365 Copilot APIs (use API directly)
  • Need multi-agent orchestration (use Agent Framework)
  • Have existing Bot Framework bot (consider DirectLine)

Key Capabilities

  • Deploy to Teams, M365 Copilot, Copilot Studio, Web
  • AI-agnostic architecture
  • Scaffolding for agent communication patterns
  • TypeScript and .NET support

Microsoft Agent 365 SDK

Requires Frontier Preview Access

When to Use ‚úì

  • Agent needs to respond to Teams/Outlook notifications
  • Require governed access to Mail, Calendar, SharePoint, Teams
  • Need enterprise identity and security
  • Want admin-controlled data access policies
  • Require Model Context Protocol (MCP) integration

When NOT to Use ‚úó

  • Don't need M365 data access (use M365 Agents SDK)
  • Building standalone bot (use DirectLine API)
  • Only need Copilot APIs (use M365 Copilot API)
  • Can't access Frontier preview program

Enterprise Features

  • Notifications from Teams, Outlook, Word, Email
  • Governed MCP servers for M365 workloads
  • Agent 365 CLI for development lifecycle
  • Enterprise security and identity management
  • Admin-controlled access policies

Microsoft Agent Framework

When to Use ‚úì

  • Need multiple AI agents working together
  • Building complex agentic workflows
  • Want cloud-agnostic deployment
  • Need to connect REST APIs as tools (OpenAPI)
  • Want Model Context Protocol (MCP) integration
  • Need local development with Azure deployment option

When NOT to Use ‚úó

  • Building single-agent chatbot (use M365 Agents SDK)
  • Only need API access (use M365 Copilot API)
  • Want fully managed service (use Foundry Agent Service)
  • Prefer no-code/low-code solutions

Core Capabilities

  • Multi-agent orchestration (AutoGen + Semantic Kernel)
  • OpenAPI-first tool integration
  • Cloud-agnostic runtime (containers, on-prem, multi-cloud)
  • Model Context Protocol support
  • Dynamic tool connections

Azure AI Foundry SDK & Foundry Agent Service

When to Use ‚úì

  • Building AI apps on Azure infrastructure
  • Need Azure OpenAI, model catalog, prompt flow
  • Want integrated observability and evaluation
  • Require enterprise compliance and governance
  • Need to deploy agents without infrastructure management
  • Want to use LangGraph, CrewAI, or Agent Framework

When NOT to Use ‚úó

  • Need M365-specific features (use M365/Agent 365 SDK)
  • Building simple bots (use DirectLine API)
  • Require on-premises deployment
  • Want to avoid cloud vendor lock-in

Platform Features

  • Azure AI model catalog and Azure OpenAI
  • Prompt flow and evaluation tools
  • Built-in observability and monitoring
  • Enterprise security and compliance
  • Managed agent hosting (Foundry Agent Service)
  • Support for multiple agent frameworks
